Para probarlo:

1. Situate en la carpeta en la que esten los ficheros fuente.
2. Una vez estes en esa carpeta, crea una carpeta llamada bin (o como quieras),
   que sera donde se guardaran los .class
3. Compila los fuentes mediante el comando:

		javac -d bin *.java
		
   Si la carpeta para los .class la has llamado distinta a bin, pues cambia el nombre
   al ejecutar el comando con el que le hayas puesto
4. Una vez se hayan compilado los fuente, ve a la carpeta donde esten los .class, en mi
   caso es la carpeta bin   ->  cd bin
5. Ejecuta el comando rmiregistry . Lo tienes que ejecutar en la carpeta donde se encuentren
   los .class, si no, no funcionara.
6. Abre una nueva ventana de la linea de comandos y situate en la carpeta donde esten los
   ficheros fuente.
7. Lanza tantos servidores de calculo (WorkerServer) como quieras mediante el comando:

		java -classpath bin -Djava.rmi.server.codebase=file:bin/ LanzarWorker &
		
8. Lanza el servidor de asignacion (WorkerFactoryServer) mediante el comando:

		java -classpath bin -Djava.rmi.server.codebase=file:bin/ LanzarFactory &
		
9. Y por ultimo, lanza el cliente mediante el comando:

		java -classpath bin Cliente [min] [max] [servsCalc]
	
	Sustituyendo los parametros min, max y servsCalc por numeros enteros.
